# Plan: Add Release Please for Automated CLI Releases

## Context

The `rudel` CLI (published to npm as `rudel`, currently v0.1.4) is released manually via `scripts/release-cli.ts`, which requires interactive npm OTP and running the script locally. The goal is to automate changelog generation from conventional commits and automate npm publishing when a Release PR is merged.

## Changes

### 1. Create `release-please-config.json` (new file, root)

Release Please manifest configuration. Only `apps/cli` is releasable. Tag format matches existing `rudel@X.Y.Z` convention.

```json
{
  "$schema": "https://raw.githubusercontent.com/googleapis/release-please/main/schemas/config.json",
  "include-v-in-tag": false,
  "tag-separator": "@",
  "packages": {
    "apps/cli": {
      "release-type": "node",
      "component": "rudel",
      "changelog-path": "CHANGELOG.md",
      "extra-files": [
        { "type": "generic", "path": "src/app.ts" }
      ]
    }
  }
}
```

### 2. Create `.release-please-manifest.json` (new file, root)

Tracks current version. Initialized to match the latest published version.


### 3. Annotate `apps/cli/src/app.ts` (line 29)

Add `// x-release-please-version` so Release Please auto-updates the hardcoded version string alongside `package.json`.

### 4. Create `.github/workflows/release.yml` (new file)

Two jobs:
- **release-please**: Runs on push to `main`. Creates/updates a Release PR with changelog + version bump. When the Release PR is merged, creates a GitHub Release + tag.
- **publish-cli**: Conditional on `release_created`. Checks out, installs, builds, and runs `bun publish` with `NPM_CONFIG_TOKEN`.

Mirrors existing CI patterns (Bun 1.3.5, cache setup, frozen lockfile).

### 5. Create `.github/workflows/pr-title.yml` (new file)

Uses `amannn/action-semantic-pull-request@v5` to enforce conventional commit format on PR titles. Since GitHub squash-merge uses the PR title as the commit message, this is sufficient without full commitlint setup.

Allowed types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`.

### 6. Update `CLAUDE.md`

Add a "Releasing" section documenting the automated flow and keeping the manual script as a documented fallback.

## Manual Step (not automated by code)

**Add `NPM_TOKEN` GitHub secret**: Generate an npm Automation-type granular access token (bypasses 2FA) scoped to the `rudel` package, and add it as `NPM_TOKEN` in the repo's GitHub Actions secrets.

## How it works end-to-end

1. PRs must have conventional commit titles (enforced by `pr-title.yml`)
2. Merged PRs trigger `release.yml` — Release Please creates/updates a Release PR accumulating changes
3. Merging the Release PR triggers `release.yml` again — Release Please creates a GitHub Release + tag, then `publish-cli` publishes to npm
