### Key Patterns

- **Contract-First RPC**: The API contract in `@rudel/api-routes` is the single source of truth. The web app imports the contract type and creates a fully typed client. No manual endpoint URLs or response parsing.
- **URL + localStorage State Persistence**: Both DateRange and Filter contexts sync state to URL search params (for shareability) and localStorage (for persistence across navigations).
- **Callback-Based Data Flow**: The SessionDetailPage orchestrates multiple child components (ConversationView, TokenUsageChart, ToolActivityChart) through callbacks rather than shared state or context. ConversationView parses data and pushes chart data up; charts push click events up; the parent coordinates scroll-to-message actions back down.
- **IntersectionObserver for Scroll Tracking**: ConversationView uses an IntersectionObserver with a -80% bottom root margin to determine which message is currently "active" as the user scrolls, feeding this to the chart cursor.
- **Dual Schema Support**: The conversation parser handles both role-based (`{ role: "user", content: ... }`) and type-based (`{ type: "user", message: { content: ... } }`) JSONL entry formats, accommodating different data sources.
- **Custom SVG Charts**: The TokenUsageChart and ToolActivityChart are hand-drawn SVG (no charting library), while the analytics page charts use Recharts.</result>
