# Plan: Context-aware empty state on Dashboard

## Context

When the Dashboard shows zero sessions for the selected time range, it always displays `CliSetupHint` (install CLI instructions). This is wrong when the user has sessions — just not in the selected range. We need to distinguish between "no sessions ever" (show CLI setup) and "no sessions in this range" (show a hint to change the time filter).

## Approach

Add a `total_sessions` field to the existing KPIs response so the frontend can distinguish the two states. On the frontend, create a lightweight empty-range component with a "View all time" button.

## Changes

### 1. API contract — add `total_sessions` to KPIs schema
**File:** `packages/api-routes/src/schemas/analytics.ts`
- Add `total_sessions: z.number()` to `OverviewKPIsSchema`

### 2. API service — query total sessions without date filter
**File:** `apps/api/src/services/overview.service.ts`
- In `getOverviewKPIs`, add a subquery `(SELECT count() FROM rudel.session_analytics WHERE organization_id = '...') as total_sessions` (no date filter)
- Include `total_sessions` in the returned object and default value

### 3. Frontend — new `NoSessionsInRange` component
**File:** `apps/web/src/components/analytics/NoSessionsInRange.tsx` (new)
- Shows "No sessions found" heading with a message about the selected time range
- "View all time" button that sets the date range to cover all history (start = `2024-01-01`, end = today)
- Uses `useDateRange()` to set dates programmatically
- Styled consistently with `CliSetupHint` (uses `AnalyticsCard`, similar layout)

### 4. Frontend — update `OverviewPage` rendering logic
**File:** `apps/web/src/pages/dashboard/OverviewPage.tsx`
- Change the empty-state decision:
  - `kpis.total_sessions === 0` → render `CliSetupHint`
  - `kpis.distinct_sessions === 0 && kpis.total_sessions > 0` → render `NoSessionsInRange`
- Show the `DatePicker` in the header even when `distinct_sessions === 0` but `total_sessions > 0` (so the user can also change the range from the header)

## Verification

1. Run `bun run verify` (typecheck + lint + tests)
2. Manual test on `localhost:4011/dashboard`:
   - With data in range → full dashboard
   - With data but not in selected range → "No sessions in range" + "View all time" button
   - With no data at all → CLI setup hint
